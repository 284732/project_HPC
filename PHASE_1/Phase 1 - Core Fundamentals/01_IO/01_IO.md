# Input/Output in C++

## Introduction

A scientific program almost always needs to read input data and write results:
configuration parameters, experimental data, numerical output, execution logs. These
operations are called **input/output** (I/O).

C++ handles I/O through the **stream** library (*iostreams*). A stream is an abstraction of
a flow of data between the program and a source or destination (keyboard, screen, file).
Standard I/O lives in the `<iostream>` header, file I/O in `<fstream>`, and the format
manipulators in `<iomanip>`. Unlike C, which uses functions such as `printf`/`scanf` with
format strings, C++ uses the **overloaded operators** `<<` (insertion, output) and `>>`
(extraction, input), which are *type-safe*: the type of the data automatically determines
how it is read or written.

## 1. The standard streams

At the start of every C++ program a few predefined streams are available, declared in the
`std` namespace:

| Stream      | Meaning                              | Typical use                          |
| ----------- | ------------------------------------ | ------------------------------------ |
| `std::cin`  | standard input                       | reading data (usually the keyboard)  |
| `std::cout` | standard output                      | printing results (usually the screen)|
| `std::cerr` | standard error (unbuffered)          | error messages                       |
| `std::clog` | standard error (buffered)            | logs and diagnostics                 |

The distinction between `cout` and `cerr` matters in HPC: by separating the results output
from the error output, the two flows can be redirected to different files
(`./program > results.txt 2> errors.txt`).

## 2. Output with `std::cout` and `<<`

The `<<` operator "inserts" a value into the stream. Several insertions can be chained in
the same statement:

```cpp
#include <iostream>

int n = 42;
double x = 3.14159;
std::cout << "n = " << n << ", x = " << x << "\n";
```

No format specifiers such as `%d` or `%f` are needed: the type of the variable determines
how it is printed. The `std::endl` manipulator inserts a newline **and** forces the buffer
to be flushed; `"\n"` is often preferred, as it does not force a flush and is therefore
more efficient inside loops.

### Format manipulators (`<iomanip>`)

To control width, alignment and precision — useful for aligned numerical tables — the
manipulators are used:

```cpp
#include <iomanip>

std::cout << std::fixed << std::setprecision(3);   // 3 decimal digits, fixed notation
std::cout << std::setw(8) << x << "\n";            // minimum width of 8 columns
```

This is the "object-oriented" equivalent of C's format descriptors.

## 3. Input with `std::cin` and `>>`

The `>>` operator "extracts" a value from the stream, automatically skipping leading
whitespace. The `&` operator is **not** needed: C++ uses references, so the variable is
filled directly.

```cpp
int n;
double x;
std::cin >> n >> x;     // reads an integer and a double
```

Extraction returns the stream itself, which can be evaluated as a boolean condition: it is
*true* if the read succeeded. This is the correct idiom for detecting malformed input:

```cpp
if (!(std::cin >> n)) {
    std::cerr << "Invalid input\n";
}
```

## 4. Strings and reading lines

C++ provides the `std::string` type (header `<string>`), much safer and more convenient
than C's `char` arrays. To read a whole line, including spaces, `std::getline` is used:

```cpp
#include <string>
std::string line;
std::getline(std::cin, line);
```

To parse the fields of a line, a `std::istringstream` can be used (header `<sstream>`),
the equivalent of `sscanf`:

```cpp
#include <sstream>
std::istringstream iss(line);
double a, b;
iss >> a >> b;
```

## 5. File I/O

Files are handled with the classes from `<fstream>`:

- `std::ofstream` for writing;
- `std::ifstream` for reading;
- `std::fstream` for both.

Opening a file amounts to constructing the stream object with the file name; closing is
automatic when the object goes out of scope (RAII), or explicit with `close()`:

```cpp
#include <fstream>

std::ofstream out("data.txt");      // opens for writing
if (!out) {                         // check: did the open fail?
    std::cerr << "Cannot open the file\n";
    return 1;
}
out << "result " << 42 << "\n";     // written just like to cout
// out is closed automatically at end of scope
```

Reading is symmetric with `std::ifstream` and the `>>` operators or `std::getline`.

### Open modes

The `std::ios` flags can be combined with the `|` operator:

| Flag              | Meaning                                       |
| ----------------- | --------------------------------------------- |
| `std::ios::in`    | open for reading                              |
| `std::ios::out`   | open for writing (truncates, default for ofstream) |
| `std::ios::app`   | append: always writes at the end              |
| `std::ios::trunc` | clears the file on opening                     |
| `std::ios::binary`| binary mode instead of text mode              |

### Binary I/O

For large amounts of numerical data, **binary** I/O is often preferred: it writes the
bytes exactly as they are in memory, without converting them to text. The `write` and
`read` methods are used, working on a pointer to `char`:

```cpp
std::ofstream out("data.bin", std::ios::binary);
out.write(reinterpret_cast<const char*>(v.data()), v.size() * sizeof(double));
```

Binary is more compact and faster than text (no number↔string conversion), but the file is
not directly readable and depends on the machine representation (*endianness*, type
sizes). It is a common choice in HPC when I/O performance matters.

## 6. I/O error handling

Every stream has a **state** that can be queried with dedicated methods:

- `good()` — no error;
- `eof()` — end of file reached;
- `fail()` — logical error (e.g. a failed conversion) or open failure;
- `bad()` — serious I/O error.

The stream is also convertible to `bool` (`if (stream) ...`), which allows compact idioms
such as `while (in >> value) { ... }`, reading as long as extraction succeeds.

Alternatively, streams can be configured to **throw exceptions** with the `exceptions(...)`
method, integrating with C++'s `try`/`catch` mechanism. This is an advantage over C, which
has no exceptions and relies only on checking return values.

## 8. Summary

I/O in C++ is based on streams and the type-safe operators `<<` and `>>`. The standard
streams `std::cin`, `std::cout` and `std::cerr` are always available; files are handled
with `std::ofstream`/`std::ifstream`, closed automatically thanks to RAII. The `<iomanip>`
manipulators control the numerical format, while binary I/O (`write`/`read`) is more
efficient for large datasets. Robustness is achieved by checking the stream state
(`good`, `eof`, `fail`, `bad`) or, alternatively, through exceptions. Compared with C, C++
offers I/O that is more type-safe and more integrated with error handling.
