# Input/Output in C

## Introduzione

Un programma scientifico ha quasi sempre bisogno di leggere dati di ingresso e di
scrivere risultati: parametri di configurazione, dati sperimentali, output numerici,
log di esecuzione. Queste operazioni vengono chiamate **input/output** (I/O).

In C tutte le funzionalità di I/O sono fornite dalla libreria standard, in particolare
dall'header `<stdio.h>` (*standard input/output*). Il modello del C si basa sul concetto
di **flusso** (*stream*): un canale di byte che collega il programma a una sorgente o a
una destinazione di dati (tastiera, schermo, file su disco). A differenza del Fortran,
che associa i file a delle *unit* numeriche, il C rappresenta ogni file aperto con un
puntatore a una struttura di tipo `FILE`.

## 1. I flussi standard

All'avvio di ogni programma C sono disponibili tre flussi predefiniti:

| Flusso   | Significato            | Uso tipico                          |
| -------- | ---------------------- | ----------------------------------- |
| `stdin`  | standard input         | lettura dei dati (di solito tastiera) |
| `stdout` | standard output        | stampa dei risultati (di solito schermo) |
| `stderr` | standard error         | messaggi di errore e diagnostica    |

La distinzione tra `stdout` e `stderr` è importante in ambito HPC: separando l'output
dei risultati da quello degli errori, è possibile redirezionare i due flussi su file
diversi (`./programma > risultati.txt 2> errori.txt`).

## 2. Output formattato: `printf`

La funzione `printf` stampa su `stdout` secondo una *stringa di formato* che contiene
testo normale e **specificatori di conversione** introdotti dal carattere `%`.

```c
printf("Il valore di pi greco è circa %f\n", 3.14159);
```

Gli specificatori più comuni sono:

| Specificatore | Tipo                         |
| ------------- | ---------------------------- |
| `%d` / `%i`   | intero con segno (`int`)     |
| `%u`          | intero senza segno           |
| `%f`          | numero in virgola mobile (`double`/`float`) |
| `%e`          | virgola mobile in notazione esponenziale |
| `%g`          | sceglie automaticamente `%f` o `%e` |
| `%c`          | singolo carattere            |
| `%s`          | stringa (array di `char`)    |
| `%p`          | puntatore (indirizzo)        |
| `%%`          | il carattere `%` letterale   |

È possibile controllare ampiezza e precisione, cosa molto utile per produrre tabelle
numeriche allineate: `%8.3f` stampa un numero in almeno 8 colonne con 3 cifre decimali.
Questo livello di controllo è concettualmente simile ai descrittori `FORMAT` del Fortran.

## 3. Input formattato: `scanf`

La funzione `scanf` legge da `stdin` interpretando i dati secondo una stringa di formato
analoga a quella di `printf`. Un aspetto **fondamentale e tipico del C** è che a `scanf`
si deve passare l'**indirizzo** della variabile in cui memorizzare il valore, usando
l'operatore `&`:

```c
int n;
scanf("%d", &n);   /* &n è l'indirizzo di n */
```

Il motivo è che in C i parametri sono sempre passati *per valore*: per permettere a
`scanf` di modificare la variabile del chiamante, occorre passarle dove si trova in
memoria (il suo puntatore). Questo concetto verrà ripreso nel modulo sulle procedure.

`scanf` restituisce il numero di elementi letti correttamente: controllare questo valore
di ritorno è il modo corretto per accorgersi di un input malformato.

## 4. I/O carattere e per riga

Oltre all'I/O formattato esistono funzioni più elementari:

- `getchar()` / `putchar(c)` leggono e scrivono un singolo carattere;
- `fgets(buffer, dim, stream)` legge un'intera riga in modo sicuro, senza superare la
  dimensione del buffer;
- `fputs(stringa, stream)` scrive una stringa.

In C moderno si preferisce sempre `fgets` alla vecchia funzione `gets`, che è stata
**rimossa dallo standard** perché non controlla la dimensione del buffer ed è quindi
fonte di gravi vulnerabilità (buffer overflow).

## 5. I/O su file

Per lavorare con un file su disco occorre prima aprirlo con `fopen`, che restituisce un
puntatore `FILE *`:

```c
FILE *f = fopen("dati.txt", "r");
if (f == NULL) {
    perror("Impossibile aprire dati.txt");
    return 1;
}
/* ... operazioni sul file ... */
fclose(f);
```

Le modalità di apertura principali sono:

| Modalità | Significato                                                       |
| -------- | ---------------------------------------------------------------- |
| `"r"`    | lettura; il file deve esistere                                    |
| `"w"`    | scrittura; crea il file o ne **azzera** il contenuto se esiste    |
| `"a"`    | append; scrive in coda al file, creandolo se non esiste           |
| `"r+"`   | lettura e scrittura su file esistente                             |
| `"w+"`   | lettura e scrittura, troncando il file                            |
| `"a+"`   | lettura e append                                                  |

Aggiungendo `b` (es. `"rb"`, `"wb"`) si apre il file in modalità **binaria** anziché
testuale.

Le funzioni di I/O su file sono le analoghe "f" di quelle standard:
`fprintf`/`fscanf` per l'I/O formattato, `fputs`/`fgets` per le righe,
`fwrite`/`fread` per blocchi di byte (I/O binario). Di fatto, `printf(...)` equivale a
`fprintf(stdout, ...)`.

### I/O binario

Per grandi quantità di dati numerici si preferisce spesso l'I/O **binario**, che scrive
i byte così come sono in memoria, senza convertirli in testo:

```c
fwrite(vettore, sizeof(double), n, f);   /* scrive n double */
fread (vettore, sizeof(double), n, f);   /* li rilegge      */
```

Il binario è più compatto e veloce del testo (niente conversione numero↔stringa), ma il
file non è leggibile direttamente e dipende dalla rappresentazione della macchina
(*endianness*, dimensione dei tipi). È una scelta molto comune in HPC quando le
prestazioni di I/O contano.

## 6. Gestione degli errori di I/O

Il C non ha un meccanismo di eccezioni come Python: la gestione degli errori si basa sul
**controllo dei valori di ritorno** e su alcune funzioni dedicate:

- `fopen` restituisce `NULL` in caso di fallimento;
- le funzioni di lettura restituiscono `EOF` (o un conteggio inferiore al richiesto)
  alla fine del file o in caso di errore;
- `feof(f)` indica se è stata raggiunta la fine del file;
- `ferror(f)` indica se si è verificato un errore di I/O;
- `perror("messaggio")` stampa un messaggio seguito dalla descrizione testuale
  dell'ultimo errore di sistema (basata sulla variabile `errno`).

Controllare sempre questi valori è considerato buona pratica: un programma robusto deve
reagire a file mancanti o a dati malformati senza terminare in modo incontrollato.

## 8. Riepilogo

L'I/O in C è gestito dalla libreria standard `<stdio.h>` attraverso il concetto di
flusso. I flussi standard `stdin`, `stdout` e `stderr` sono sempre disponibili; i file
si manipolano tramite un puntatore `FILE *` ottenuto con `fopen` e rilasciato con
`fclose`. L'I/O formattato (`printf`/`scanf` e le varianti `f...`) offre un controllo
fine sul formato, simile ai descrittori del Fortran, mentre l'I/O binario
(`fwrite`/`fread`) è più efficiente per grandi dataset numerici. In assenza di
eccezioni, la robustezza si ottiene controllando sistematicamente i valori di ritorno e
usando funzioni come `feof`, `ferror` e `perror`.
